# GitHub Setup

Suggested repository name:

```text
perturbflow-analyzer
```

Suggested first workflow:

1. Initialize the local repository.
2. Commit the package and documentation scaffold.
3. Create a GitHub remote repository.
4. Push a feature branch.
5. Open a pull request for review.

When GitHub Pages is enabled, publish the MkDocs site from the `site/` build artifact or a documentation deployment workflow.
