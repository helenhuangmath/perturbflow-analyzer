# Vision

PerturbFlow is **open infrastructure for perturbation biology**. Rather than
being one more standalone analysis method, it standardizes how perturbation data
is represented, analyzed, interpreted, and evaluated — connecting experimental
data, mechanistic interpretation, and scalable workflows in one interoperable,
AnnData-native framework that complements the scverse ecosystem.

A distinctive capability is **rewiring-aware interpretation**: distinguishing
perturbations that *amplify* existing transcriptional programs from those that
*reorganize* regulatory relationships — interpretable biological signal that
prediction-accuracy metrics alone cannot capture.

## Three aims

### Aim 1 — Community-standard infrastructure

Stable AnnData schemas, reproducible workflows, interactive reports, reference
datasets, and interoperable APIs.

*Milestone:* a `v1.0` release with validated workflows across at least three
public perturbation datasets.

### Aim 2 — Model-ready perturbation biology

Standardized exports, scalable processing for large screens, and interfaces for
workflow managers.

*Milestone (month 18):* a public benchmark-ready dataset collection and an
interoperability toolkit.

### Aim 3 — Community benchmarks

Baseline-calibrated evaluation, biological-distance-aware diagnostics,
rewiring-aware metrics, and reproducible benchmarking workflows — so that claims
of improvement are measured against honest baselines.

*Milestone (month 24):* a public benchmark suite and evaluation portal.

## Where the project stands

PerturbFlow has been applied to large-scale public Perturb-seq datasets and is
in active development.
